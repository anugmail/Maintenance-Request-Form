## 🎯 ขอบเขตงานนี้

สร้างต้นแบบ (mock) ฟอร์มแจ้งซ่อมที่คลิกเล่นได้จริง พร้อมเอกสารอ้างอิง design เพื่อใช้คุยกับทีม/ลูกค้า ก่อนแตกการ์ดเทคนิค

**Flow ที่ยืนยันแล้ว (17 ก.ค. 2569):**

```
① เลือกรถ (ข้อมูลรถ/อุปกรณ์เสริมมาเอง)
② เลือกอาการเสีย (รายการมาตรฐานตามหมวด + อื่นๆ + ความรุนแรง + แนบรูป)
③ ระบบ Suggest อะไหล่ (mock) + แสดงคงเหลือ/สถานะคลัง → จองอะไหล่ที่มีของได้
④ ตัดสินใจ: ซ่อมเอง หรือ ส่งซ่อม กบค. → สรุป + ส่งเรื่อง → ได้เลขที่ใบแจ้งซ่อม
```

จุดสำคัญ: **ผู้แจ้งเห็นอะไหล่+ของคงเหลือก่อน แล้วค่อยตัดสินใจ** ว่าซ่อมเองหรือส่ง กบค. — สอดคล้องกับ [US-08](../backlog/BL01-แจ้งซ่อม-Suggest-อะไหล่/US-08-บันทึกการตัดสินใจ.md)

## 📝 บันทึกการตัดสินใจ (17 ก.ค. 2569)

1. **ตำแหน่งงาน:** แยกโฟลเดอร์ `Maintenance-Request/Maintenance-Request-Form/` (พี่น้องกับ backlog/BL01) — *เดิมชื่อ `รถก่อสร้าง/ฟอร์มแจ้งซ่อม/` เปลี่ยนเป็นภาษาอังกฤษ 17 ก.ค. 2569 พร้อมไฟล์ mock → `Maintenance-Request-Form.html`*
2. **Deliverable:** HTML mock คลิกได้จริง + เอกสาร design reference
3. **Flow:** เลือกอาการเสียก่อน → เห็นอะไหล่แนะนำ+สต็อก → ค่อยเลือก **ซ่อมเอง / ส่ง กบค.** (แก้จาก draft แรกที่ให้เลือกวิธีซ่อมก่อนเลือกอาการ)
4. **การจองอะไหล่:** จองได้จากหน้าอะไหล่แนะนำเมื่อคลังมีของ (mock — การเบิก/ตัดสต็อกจริงนอกขอบเขต BL01)
5. Suggest **อะไหล่อย่างเดียว** ไม่แนะนำวิธีซ่อม (ตามมติ BL01 — US-05 ตัดออก)

## ⚠️ ประเด็นเปิด / สิ่งที่ต้องยืนยันต่อ

| # | ประเด็น | หมายเหตุ |
|---|---|---|
| 1 | ช่องทาง "อู่" หายไปจาก flow ล่าสุด — ตัวเลือกตัดสินใจตอนนี้คือ ซ่อมเอง/ส่ง กบค. | ถ้าต้องมี "ส่งอู่" เพิ่ม เป็นตัวเลือกที่ 3 ของขั้นตัดสินใจ หรือเป็นทางแยกของ "ซ่อมเอง"? ยืนยันกับเจ้าของงาน |
| 2 | Routing ตามสังกัด (เขต→กรย. / สนญ.→กบค. ตาม US-08) | Mock นี้ fix ปลายทางเป็น กบค. ไปก่อน — ของจริงต้องตั้งค่า routing ได้ |
| 3 | เกณฑ์ "ใกล้หมด" ใน mock ใช้ คงเหลือ ≤ 3 | เกณฑ์จริงรอคำตอบคำถามเปิด #3 ของ BL01 |

## 🔐 ผลการเข้าระบบ VMS Plus dev (17 ก.ค. 2569)

- บัญชี `700001` **ผ่าน PEA SSO (Keycloak) ได้** — sso2.pea.co.th คืน authorization code ปกติ
- แต่ backend VMS Plus dev ตอบ **401 ที่ `POST /api/login/authen-keycloak`** พร้อม toast "คุณไม่มีสิทธิเข้าใช้งานระบบ" → **บัญชีนี้ยังไม่ถูกเปิดสิทธิ์ในระบบ VMS Plus (dev)** ต้องให้ผู้ดูแลระบบเปิดสิทธิ์ก่อนจึงจะเข้าหน้า request-list ได้
- ช่องทาง "บุคคลภายนอก" (`/login-os`) เป็น login ด้วย เบอร์โทร+OTP/ThaID สำหรับพนักงานขับรถ — ใช้ username/password ไม่ได้
- design reference จึงสกัดจาก: หน้า landing, หน้า SSO, หน้า login-os และ CSS bundle สาธารณะของแอป (ดู 01-Design-Reference)
