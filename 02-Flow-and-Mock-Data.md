# 🔁 Flow ฟอร์มแจ้งซ่อม + Mock Data

> Flow ยืนยันกับเจ้าของงานแล้ว 17 ก.ค. 2569: **เลือกอาการเสียก่อน → เห็นอะไหล่ → ค่อยตัดสินใจ ซ่อมเอง/ส่ง กบค.**

## Flow ละเอียด

```mermaid
flowchart TD
    A([เปิดฟอร์มแจ้งซ่อม]) --> B[① เลือกรถ<br/>ค้นจากทะเบียน/รถของฉัน<br/>ระบบดึง ยี่ห้อ รุ่น สังกัด อุปกรณ์เสริม ให้เอง]
    B --> B2[ระบุจุดที่มีปัญหา: ตัวรถ / อุปกรณ์เสริม]
    B2 --> C[② เลือกอาการเสีย<br/>รายการมาตรฐานตามหมวด กรองตามจุดที่เลือก<br/>+ อื่นๆ ระบุเอง + ความรุนแรง + คำอธิบาย + แนบรูป]
    C --> D[③ ระบบ Suggest อะไหล่ ตามอาการ mock<br/>แสดง จำนวนที่ต้องใช้ / คงเหลือ / สถานะคลัง / อัปเดตเมื่อ]
    D --> D2{คลังมีของ?}
    D2 -- มี --> D3[จองอะไหล่ได้ เลือกจำนวน]
    D2 -- หมด/รอของ --> D4[เห็นสถานะ หมด หรือ รอของ พร้อม ETA]
    D3 --> E
    D4 --> E[④ ตัดสินใจ]
    E -- ซ่อมเอง --> F[กรอกแผน/บันทึกการซ่อมโดยย่อ<br/>ส่งเรื่อง → สถานะ บันทึกแล้ว-ซ่อมเอง]
    E -- ส่งซ่อม กบค. --> G[กรอกเหตุผลที่ซ่อมเองไม่ได้<br/>เห็นหน่วยงานปลายทาง กบค. ก่อนยืนยัน<br/>ส่งเรื่อง → สถานะ ส่งเรื่องแล้ว รอ กบค. รับเรื่อง]
    F --> H([ได้เลขที่ใบแจ้งซ่อม MTD-YYMMDD-XXX<br/>แนบข้อมูล อาการ รูป อะไหล่ที่แนะนำ/จอง ติดเรื่องไว้เสมอ])
    G --> H
```

**กติกาใน mock:**

- อะไหล่แนะนำแสดง **ทันทีที่เลือกอาการ** (ขั้น ③ อัปเดตตามอาการที่ติ๊ก) — แสดงคู่กับสต็อกจอเดียวกัน (มติ BL01: ไม่แยกขั้น)
- จองได้เฉพาะรายการที่คงเหลือ > 0 · จำนวนจองเริ่มต้น = จำนวนที่ต้องใช้ แต่ไม่เกินคงเหลือ
- เปลี่ยนอาการ → รายการอะไหล่อัปเดต และการจองของอาการที่เอาออกถูกยกเลิก (แจ้งผู้ใช้)
- สถานะคลัง: `พร้อมเบิก` (>3) · `ใกล้หมด` (1–3) · `หมด` (0) · `หมด-รอของ` (0 + มี ETA) — เกณฑ์ "ใกล้หมด" เป็นค่าสมมุติ รอคำตอบคำถามเปิด #3
- ใบแจ้งซ่อมเก็บ snapshot รายการอะไหล่ที่ระบบแนะนำไว้กับเรื่องเสมอ (AC ของ US-06/US-08)
